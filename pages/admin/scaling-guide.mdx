# Scaling Guide: Multi-Tenant iMessage Infrastructure

This guide covers how to scale TextBubbles to support multiple customers with dedicated phone numbers.

## Current Architecture

```
                    ┌─────────────────────────────────┐

                    └─────────────┬───────────────────┘

                    ┌─────────────▼───────────────────┐
                    │     BlueBubbles Router          │
                    │   (instance selection)          │
                    └─────────────┬───────────────────┘

         ┌────────────────────────┼────────────────────────┐
         │                        │                        │
```

## Scaling Options

### Option 1: Fast User Switching (Recommended for MVP+)

**How it works:** Multiple macOS user accounts on a single Mac, each running BlueBubbles.

| Metric | Value |
|--------|-------|
| Max concurrent users | **5 per Mac** (macOS limit) |
| Setup complexity | Low |
| Cost per customer | ~$100 (shared hardware) |
| Isolation | Medium (shared hardware) |

**Requirements:**
- Mac Mini M4 (24GB+ RAM recommended)
- Separate macOS user per customer
- Each user has own iCloud account
- BlueBubbles runs as login item for each user

**Setup Script:**
```bash
# Create new macOS user for customer
sudo dscl . -create /Users/customer1
sudo dscl . -create /Users/customer1 UserShell /bin/zsh
sudo dscl . -create /Users/customer1 RealName "Customer 1"
sudo dscl . -create /Users/customer1 UniqueID 510
sudo dscl . -create /Users/customer1 PrimaryGroupID 20
sudo dscl . -create /Users/customer1 NFSHomeDirectory /Users/customer1
sudo dscl . -passwd /Users/customer1 <password>
sudo createhomedir -c -u customer1

# Login as that user and configure iCloud + BlueBubbles
```

**Limitations:**
- Maximum 5 users per Mac Mini
- All users share CPU/RAM
- Requires physical access for initial iCloud setup
- iCloud may require verification for new accounts

---

### Option 2: macOS VMs (UTM/Parallels)

**How it works:** Run multiple macOS VMs, each with its own BlueBubbles instance.

| Metric | Value |
|--------|-------|
| Max VMs per Mac | **2-4** (resource dependent) |
| Setup complexity | Medium |
| Cost per customer | ~$150-250 |
| Isolation | High |

**Recommended Specs per VM:**
- 4 CPU cores
- 8GB RAM
- 64GB storage
- macOS Sonoma or newer

**VM Options:**

| Tool | License | Notes |
|------|---------|-------|
| **UTM** | Free | Uses Apple Hypervisor, near-native ARM performance |
| **Parallels** | $100/yr | Best Windows support, good macOS VM support |
| **VMware Fusion** | Free/Pro | Mature, enterprise features |

**Important Caveats:**
- iMessage activation in VMs can be tricky
- Apple may flag VM-based iCloud logins
- BlueBubbles has [specific VM guidance](https://docs.bluebubbles.app/server/advanced/macos-virtualization/running-a-macos-vm)
- UTM on Apple Silicon runs ARM macOS only (not Intel)

---

### Option 3: Dedicated Mac Hardware (Enterprise)

**How it works:** One Mac Mini per customer for maximum isolation.

| Metric | Value |
|--------|-------|
| Instances per device | 1 |
| Setup complexity | Low |
| Cost per customer | ~$500-700 |
| Isolation | Maximum |

**Recommended Hardware:**
- Mac Mini M4 (base model: $599)
- 16GB RAM (sufficient for single BlueBubbles)
- 256GB SSD

**Benefits:**
- Complete isolation
- No resource contention
- Simplest to manage at scale
- Easy to replace/troubleshoot

**Deployment:**
```
┌─────────────────────────────────────────────────────┐
│                  Colocation Rack                     │
├─────────────────────────────────────────────────────┤
│  [Mac Mini 1]  [Mac Mini 2]  [Mac Mini 3]  ...      │
│  Customer A    Customer B    Customer C             │
│  +1555...      +1666...      +1777...              │
└─────────────────────────────────────────────────────┘
```

## Database Schema

### BlueBubbles Instances Table

```sql
CREATE TABLE bluebubbles_instances (
  id UUID PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  url VARCHAR(500) NOT NULL,          -- http://192.168.0.X:1235
  password VARCHAR(255) NOT NULL,
  phone_number VARCHAR(20) NOT NULL,
  icloud_email VARCHAR(255),
  status VARCHAR(20) DEFAULT 'active', -- active, inactive, maintenance
  capabilities JSONB,
  max_customers INTEGER DEFAULT 1,
  current_customers INTEGER DEFAULT 0,
  host_type VARCHAR(50),               -- mac_user, mac_vm, dedicated
  host_info JSONB,                     -- { macUser, vmName, hostMachine }
  health_status VARCHAR(20),
  last_health_check TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Customer → Instance Mapping

```sql
-- customerNumbers table has instance_id column
ALTER TABLE customer_numbers 
ADD COLUMN instance_id UUID REFERENCES bluebubbles_instances(id);
```

## Adding a New Instance

### 1. Register in Database

```bash
curl -X POST https://api.textbubbles.com/v1/admin/instances \
  -H "Authorization: Bearer $ADMIN_API_KEY" \
  -d '{
    "name": "Customer B Mac",
    "url": "http://192.168.0.20:1235",
    "password": "bluebubbles-password",
    "phoneNumber": "+16665551234",
    "icloudEmail": "customerb@icloud.com",
    "hostType": "dedicated",
    "hostInfo": {
      "hostMachine": "mac-mini-2"
    }
  }'
```

### 2. Assign to Customer

```bash
curl -X POST https://api.textbubbles.com/v1/admin/customers/{id}/numbers \
  -H "Authorization: Bearer $ADMIN_API_KEY" \
  -d '{
    "phoneNumber": "+16665551234",
    "provider": "bluebubbles",
    "instanceId": "instance-uuid-here"
  }'
```

### 3. Verify Routing

```bash
# Send a test message
curl -X POST https://api.textbubbles.com/v1/messages \
  -H "Authorization: Bearer $CUSTOMER_API_KEY" \
  -d '{
    "to": "+1234567890",
    "content": { "text": "Test from new instance" }
  }'
```

## Health Monitoring

The system automatically monitors instance health:

```bash
# Check all instance health
curl https://api.textbubbles.com/v1/admin/instances/health \
  -H "Authorization: Bearer $ADMIN_API_KEY"
```

Instances are marked unhealthy if:
- BlueBubbles `/ping` fails
- Response time > 5 seconds
- 3 consecutive failures

Unhealthy instances are automatically excluded from routing until recovered.

## Troubleshooting

### Instance Not Responding

1. Check BlueBubbles is running on the Mac
2. Verify network connectivity (ping the IP)
3. Check BlueBubbles logs: `~/Library/Application Support/BlueBubbles/logs`
4. Restart BlueBubbles if needed

### iMessage Not Activating

1. Sign out of iCloud completely
2. Restart the Mac
3. Sign back into iCloud
4. Enable Messages in System Settings
5. Wait 5-10 minutes for activation

### Customer Messages Going to Wrong Instance

1. Check customer's assigned numbers: `GET /v1/admin/customers/:id`
2. Verify instance_id on the number assignment
3. Restart the API to reload instance configurations
